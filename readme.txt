4. Используется отдельная таблица для каждого класса, над родительским неабстрактным классом вешается аннотация @Inheritance(strategy = InheritanceType.JOINED). Если сущестует необходимость перемапить поле, по которому маппятся таблицы, можно заюзать аннотацию @PrimaryKeyJoinColumn(name = "user_id"). Преимущества - этот подход полностью отражает объектно-ориентированную структуру, посему является наиболее понятным.
